Understands the use case for Nodemon and its limitation for development.


Hvad er Nodemon?

Nodemon er et udviklingsværktøj, der automatisk genstarter en Node.js-server, når filer ændres. Dette gør udviklingen hurtigere og mere effektiv, dvs. man slipper for at skulle genstarte serveren manuelt hver gang der bliver lavet ændringer.

Installation af Nodemon

For at installere Nodemon globalt, kan man bruge følgende kommando:

npm install -g nodemon

For at installere Nodemon lokalt i et projekt:

npm install nodemon --save-dev

Brug af Nodemon

Start Node.js-app med Nodemon i stedet for node:

nodemon.cmd app.js

Eksempel med script i package.json

Man kan tilføje et script i package.json for at gøre det lettere:

    
    "scripts": {
      "dev": "cross-env PORT=8080 nodemon app.js",
      "prod": "cross-env PORT=80 ENVIRONMENT=PROD node app.js"
    }

Nu kan man starte app'en med:

npm run dev

Begrænsninger ved Nodemon

  • Ydeevne: På store projekter kan Nodemon være langsomt, da det overvåger mange filer samtidigt.
  • Filtyper: Nodemon overvåger som standard kun JavaScript- og JSON-filer. Andre filer kræver konfiguration.
