Understands type coercion and strategies to avoid it.


Type coercion

Type coercion i JavaScript er den automatiske konvertering af værdier fra en datatype til en anden, typisk udført under operationer eller sammenligninger, der involverer forskellige datatyper.

JavaScript forsøger ofte at gøre dem kompatible, men det kan føre til uventede og nogle gange logiske fejl.


Her er et par eksempler på type coercion:

    Streng- og taladdition:

  • var number = 1;
  • var string = "Mandatory " + number; // Coercion: tal til string
  • console.log(string); // Output: "Mandatory 1"
    •

    Her konverterer JavaScript tallet 1 til strengen "1" og udfører strengsammenkædning i stedet for addition.


    Boolean-konvertering:

  • if ("hello") {
  • console.log("This will run"); // Dette vil blive udført, fordi "hello" konverteres til true
  • }
  • if (0) {
  • console.log("This will not run"); // Dette vil IKKE blive udført, fordi 0 konverteres til false
  • }
    •

    JavaScript konverterer visse værdier til true eller false i logiske sammenhænge.


    Strategier til at undgå "type coercion":

    man kan anvende lighedsoperatorer === og !== i stedet == og != når man sammenligner værdier.

  • var number = 10;
  • var stringNumber = "10";
  • console.log(number == stringNumber); // true (implicit coercion)
  • console.log(number === stringNumber); // false (no coercion)
