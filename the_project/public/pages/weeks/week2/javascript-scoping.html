Understand scoping in Javascript and know to always use const if you can get away with it otherwise let.


Scoping i JavaScript:

"Scoping" i JavaScript handler om, hvordan variabler er tilgængelige i ens kode. Det definerer, hvilke dele af koden der kan se og bruge en bestemt variabel. JavaScript har to typer af scoping:

  • Function scope
  • Block scope

var har function scope, som betyder at de er tilgængelige i hele den funktion, de er erklæret i, selvom de er inde i en blok
det kunne være if-sætning eller for-løkke

Eksempel: 


  function testVarScope() {
    if (true) {
        var x = 10;
    }
    console.log(x); // x er tilgængelig her, fordi var er function Scoped
  }
  testVarScope(); // Output: 10

const og let har block scope, som betyder at de kun er tilgængelige inden for den nærmeste blok, de er erklæret i
det kunne være en blok er defineret af krøllede parenteser {}

Eksempel: 


  function testLetScope() {
    if (true) {
      let y = 20;
    }
    console.log(y); // Fejl: y er ikke defineret her, fordi let er block scoped
  }
  testLetScope(); // Fejl: ReferenceError: y is not defined

kan ses her:


  function testLetScope() {
    if (true) {
      let y = 20;
      console.log(y);
    }
  }
  testLetScope();

Anbefalinger:

    ungå at bruge var, og brug let og const i stedet

    const er IKKE en konstant, hvilket betyder at const KAN have sin værdi ændret.

    const er kun en konstant under deklarationen og skal deklareres når den bliver initializeret
