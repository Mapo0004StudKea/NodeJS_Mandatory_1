Understand advanced things about functions in Javascript


Hoisting

Hoisting betyder, at funktioner og variabler "løftes" til toppen af deres scope under kompileringen. Derfor kan man kalde en function før dens definition:

console.log(getRandomInt(0, 10));
  function getRandomInt(min, max) {
    return Math.floor(Math.random() * (max + 1 - min) - min);
  }

Anonyme Funktioner

En anonym funktion er en funktion uden navn. De bruges ofte som værdier, der kan tildeles variabler:

const getRandomIntAnonymousFunction = function (min, max) {
  return Math.floor(Math.random() * (max + 1 - min) - min);
};

Arrow Functions

Arrow functions er en kortere syntaks for at skrive anonyme funktioner. De er især nyttige til korte funktioner:

const getRandomIntArrowFunction = (min, max) => {
  return Math.floor(Math.random() * (max + 1 - min) - min);
};

Callback Funktioner

En callback er en funktion, der gives som argument til en anden funktion og kan kaldes senere:

function genericPerformer(name, action) {
  return action(name);
}

const codingAction = (name) => `${name} likes coding..`;

console.log(genericPerformer('Lasse', codingAction)); // "Lasse likes coding.."

Callback som one-liners

Man kan bruge arrow functions som one-liners direkte i callbacks:

console.log(genericPerformer('Tara', (name) => `${name} owns at boxing`)); // "Tara owns at boxing"

Praktisk Eksempel

Her er et eksempel, hvor der bliver brugt genericPerformer til at vise forskellige handlinger:

function sleepingAction(name) {
  return `${name} loves sleeping`;
}

console.log(genericPerformer('Andreas', sleepingAction)); // "Andreas loves sleeping"

Dette eksempel viser, hvordan man kan genbruge funktioner og håndtere forskellige logikker med callback-mønsteret.
